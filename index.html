What is a Linear SVM?

Have you ever wondered how banks detect fraudulent transactions or how Gmail classifies spam? That’s the magic of Support Vector Machines (SVMs).

In simple terms, SVMs draw a decision boundary that helps machines separate data and make decisions. Let’s see this in action with baseball pitch types!

The Problem

We’re looking at four pitch types:

  • Four-Seam Fastball (FF)
  • Cutter (FC)
  • Changeup (CH)
  • Slider (SL)

Given only Release Speed and Spin Rate, can a machine (with no baseball knowledge) predict the pitch type?

The Visualization

Below is the SVM-trained decision boundary. Try switching pitch pairs to see how the boundary changes for each combination.

Prediction Example

If a new data point has a speed of 87 mph and spin rate of 2500 rpm, and the pitch pair is CH & SL, the machine will likely classify it as SL.

Limitations of SVM

In some pitch pairs, no clear boundary is drawn — the data is too close to separate cleanly. This explains occasional misclassifications, like a bank wrongly flagging a legit transaction as fraud.

Key Takeaway

This project shows how vital the decision boundary is. Without it, the machine can’t make decisions — and this logic powers spam filters, fraud detection, and much more.

The Math Behind SVM

SVMs aim to maximize the margin between decision boundaries and training data. Here’s the math:

  • Hard SVM: \( \vec{w}^\star = \sum_{i \in S} y_i a_i \text{Aug}(\vec{x}^{(i)}) \)
  • Soft SVM: \( \min_{\vec{w}, \vec{\epsilon}} \lVert{\vec{w}}\rVert^2 + C \sum \epsilon_i \)

Where \( C \) controls the tradeoff between margin size and misclassification tolerance.
